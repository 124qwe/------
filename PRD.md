1. 产品需求文档 (PRD.md)
1. 产品概述
本项目是一个基于 Web 的轻量级待办事项（Todo List）应用程序，面向个人用户。旨在帮助用户高效地管理日常任务，提供从任务创建、分类、追踪到统计的完整闭环体验。所有数据均保存在本地（LocalStorage），保护隐私且无需注册即可使用。

2. 目标用户
需要管理日常工作、学习或生活任务的个人用户，追求快速响应、无网络依赖和轻量级体验。

3. 核心功能需求
3.1 任务管理 (CRUD)
添加待办 (Create):
必填项：标题 (Title)。
选填项：描述 (Description)、截止日期 (Due Date)、优先级 (Priority: 高/中/低)、分类 (Category: 工作/生活/学习等，支持自定义或预设)。
查看任务 (Read):
以列表形式展示所有任务。
直观显示任务的关键信息（标题、状态、截止日期、优先级标签、分类标签）。
编辑待办 (Update):
支持修改已创建任务的所有字段。
支持一键切换任务状态（待办/已完成）。
删除待办 (Delete):
支持删除不需要的任务（需提供二次确认以防误删）。
3.2 筛选与搜索
多维筛选: 支持按以下维度组合筛选：
状态 (全部/待办/已完成)
优先级 (全部/高/中/低)
分类 (按现有分类过滤)
关键字搜索: 提供搜索框，实时匹配任务标题或描述中的关键字。
3.3 数据统计看板
提供一个可视化的统计区域，展示当前任务的整体情况：
任务总数
已完成数量 / 待办数量
任务完成率（百分比）
今日到期任务数（可选增强）
3.4 数据持久化
应用产生的所有数据需实时同步至浏览器 LocalStorage。
页面刷新、关闭重启后，数据及最后一次的筛选状态不会丢失。
4. UI/UX 要求
响应式设计: 适配桌面端和移动端浏览。
交互反馈: 添加、删除、完成任务时需有平滑的动画过渡和操作提示（Toast）。
视觉风格: 现代、简洁。使用 Tailwind CSS 提供的颜色系统，通过不同颜色区分优先级（如：红-高，黄-中，绿/灰-低）。
